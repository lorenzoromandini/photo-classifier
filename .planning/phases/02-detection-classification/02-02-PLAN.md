    Create photo_queue_repository.dart:
    - addToQueue(PhotoDetection): Insert or update (UPSERT) pending photo
      - Check if photo already exists by ID (path hash)
      - If exists and status is processed, skip (duplicate detection per FILE-06)
      - If new, insert with status PENDING
      - Return PhotoQueueResult (success/exists/error)
    - getPendingPhotos({int limit = 10, PhotoSource? source}): Query pending photos
      - Order by detectedAt ASC (FIFO queue)
      - Optional filter by source
      - Limit to prevent memory issues
    - getPendingCount(): Return count of PENDING photos for UI badge
    - markAsProcessing(String id): Update status to PROCESSING
    - markAsProcessed(String id, String categoryId, double confidence): Update with results
    - markAsFailed(String id, String error): Update status to FAILED with error
    - markAsSkipped(String id, String reason): Mark as skipped (corrupt, unsupported format)
    - getPhotosByStatus(PhotoStatus): Filtered query for status monitoring
    - clearOldProcessed(Duration olderThan): Cleanup processed photos older than threshold
    
    Per ERR-02: Validate file exists and is readable before adding to queue.
    Check file size > 0 to detect corrupt/incomplete writes.
  </action>
