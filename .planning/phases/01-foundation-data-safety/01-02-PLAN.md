    Create SAF wrapper with proper error handling and folder discovery:

    1. **Create SafException** class:
       - sealed class SafException : Exception()
       - data class InvalidUriException(val uri: String) : SafException()
       - data class PermissionDeniedException(val uri: String) : SafException()
       - data class FolderNotFoundException(val path: String) : SafException()

    2. **Create SafDataSource** class:
       - @Inject constructor with @ApplicationContext
       - suspend fun discoverFolders(baseUri: Uri): List<FolderInfo> - discovers folders in Pictures/
       - suspend fun countPhotos(folderUri: Uri): Int - counts image files
       - suspend fun persistFolderPermission(uri: Uri) - calls takePersistableUriPermission
       - suspend fun hasPersistablePermission(uri: Uri): Boolean - checks if permission exists
       - suspend fun listPhotos(folderUri: Uri): List<DocumentFile> - lists image files in folder

    Implementation details:
       - Use DocumentFile.fromTreeUri() for folder access
       - Skip system folders: "Android", ".thumbnails", ".trash"
       - Support image extensions: jpg, jpeg, png, webp, heic, heif (case-insensitive)
       - All operations on Dispatchers.IO
       - Handle SecurityException with PermissionDeniedException
       - Return FolderInfo data class with uri, name, displayName, photoCount
  </action>
  <verify>
    Run: `./gradlew compileDebugKotlin`
    Check: No compilation errors, imports resolve correctly
  </verify>
  <done>
    SafException sealed class created, SafDataSource has discoverFolders(), persistFolderPermission(), and listPhotos() methods
  </done>

       - suspend fun updateLearnedLabels(uri: String, labels: Map<String, Float>) - stores ML results

    3. **Create RepositoryModule** (Hilt module):
       - @Module @InstallIn(SingletonComponent::class)
       - Bind repository implementations

    Key implementation:
       - discoverAndSyncFolders does: SAF discovery → map to entities → syncFolders transaction
       - All DB operations use withContext(Dispatchers.IO)
       - Return Result<T> for error handling
       - LearningStatus enum: PENDING, IN_PROGRESS, COMPLETED
