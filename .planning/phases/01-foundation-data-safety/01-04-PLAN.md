    Create ViewModel managing onboarding flow:

    1. **OnboardingViewModel class**:
       - @HiltViewModel with injected FolderRepository, UserPreferencesRepository
       - val uiState: StateFlow<OnboardingUiState>
       - fun startOnboarding() - sets WELCOME stage
       - fun onPermissionGranted(treeUri: Uri) - persists permission, starts discovery
       - fun onPermissionDenied() - shows error, allows retry
       - private suspend fun discoverFolders(treeUri: Uri) - calls repository
       - private fun completeOnboarding() - saves to preferences, navigates

    2. **Permission handling**:
       - Uses ActivityResultContracts.OpenDocumentTree()
       - Called from UI when user taps "Grant Access"
       - On result: persist URI permission, then discover folders

    3. **Discovery flow**:
       - Set stage to DISCOVERING
       - Call folderRepository.discoverAndSyncFolders(treeUri)
       - Update uiState with discovered folder count
       - On completion: set stage COMPLETE

    4. **Error handling**:
       - Catch SafException, show user-friendly message
       - Allow retry on permission denied
       - Log errors with Timber

    Implementation notes:
       - Use viewModelScope for coroutines
       - UI state updates on Main dispatcher
       - Repository calls on IO dispatcher
  </action>
