    ```
    5. Update status: VERIFYING

    7. Update status: DELETING
    8. Delete source
    9. Update status: COMPLETED
    ```

    Error handling:

       - IOException at any step returns Result.failure
       - All operations on Dispatchers.IO

  <action>
    Create TransactionRepository for managing transaction log and recovery:

    1. **TransactionRepository class**:
       - @Inject constructor with fileOperationDao: FileOperationDao, safeFileOperations: SafeFileOperations
       - suspend fun recoverPendingOperations(): RecoveryResult - called on app startup
       - suspend fun getPendingOperations(): List<FileOperationEntity>
       - suspend fun cleanupOldOperations(olderThanDays: Int = 7) - removes completed operations
       - suspend fun getFailedOperations(): Flow<List<FileOperationEntity>>

    2. **RecoveryResult sealed class**:
       - data class Success(recovered: Int, failed: Int)
       - data class PartialSuccess(recovered: Int, failed: Int, errors: List<String>)
       - data class Failure(error: String)

    3. **Recovery logic**:
       - Query pending operations (status != COMPLETED && != FAILED)
       - For each operation:
         - If status == PENDING or COPYING: retry copy, then verify, then delete
         - If status == VERIFYING: verify copy, if valid continue to delete, else retry copy
         - If status == DELETING: retry delete
       - Track successes and failures
       - Return RecoveryResult with counts

    4. **Retry with backoff**:
       - Check retryCount before attempting
       - If retryCount >= 5, mark FAILED, don't retry
       - Otherwise increment retryCount and attempt
  </action>
  <verify>
    Run: `./gradlew compileDebugKotlin`
    Check: Repository compiles correctly
  </verify>
  <done>
    TransactionRepository can recover pending operations, cleanup old logs, track failures
  </done>
</task>
