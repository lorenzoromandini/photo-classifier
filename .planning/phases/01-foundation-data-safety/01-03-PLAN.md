    2. Create `retry_service.dart`:
       ```dart
       class RetryConfig {
         final int maxRetries;
         final Duration initialDelay;
         final Duration maxDelay;
         
         const RetryConfig({
           this.maxRetries = 5,
           this.initialDelay = const Duration(seconds: 1),
           this.maxDelay = const Duration(minutes: 5),
         });
       }
       
       class RetryService {
         Future<T> executeWithRetry<T>(
           Future<T> Function() operation, {
           RetryConfig? config,
         }) async { /* exponential backoff */ }
         
         Duration calculateDelay(int attempt, RetryConfig config) {
           // exponential backoff: initialDelay * 2^attempt
           return Duration(
             milliseconds: min(
               config.initialDelay.inMilliseconds * pow(2, attempt).toInt(),
               config.maxDelay.inMilliseconds,
             ),
           );
         }
       }
       ```

    3. Create `safe_file_service.dart`:
       - SafeFileService with dependencies: FileOperationService, TransactionRepository, RetryService
       - Future<FileResult<void>> safeMove(String sourceUri, String destUri)
         Implementation:
         ```
         1. Create transaction log (status: pending)
         2. Check storage - fail if < 100MB buffer (DATA-04)
         3. Update status: copying
         4. Call fileOperationService.copyFile with retry
         5. Update status: verifying
         6. Call fileOperationService.verifyFile
         7. Update status: deleting
         8. Call fileOperationService.deleteFile (source)
         9. Update status: completed
         ```

       - Future<StorageInfo> checkStorage() - uses getAvailableStorage
       - Future<RecoveryResult> recoverPendingOperations() - called on startup

    4. Recovery logic:
       - Query pending transactions (status != completed && != failed)
       - For each:
         - If status == pending || copying: Retry copy, verify, delete
         - If status == verifying: Verify, if valid continue delete, else retry copy
         - If status == deleting: Retry delete
         - Increment retryCount, fail if >= 5 (ERR-01)
       - Return RecoveryResult with counts
