  <files>
    app/src/main/java/com/example/photoorganizer/data/local/database/entities/CategoryEntity.kt
    app/src/main/java/com/example/photoorganizer/data/local/database/entities/FolderEntity.kt

    app/src/main/java/com/example/photoorganizer/data/local/database/entities/FileOperationEntity.kt
  </files>
  <action>
    Create four Room entities in the data/local/database/entities/ directory:

    1. **CategoryEntity** - Represents user-defined photo categories (max 10 per user constraints):
       - @PrimaryKey val id: String (UUID)
       - val name: String
       - val folderUri: String (target folder Document URI)
       - val mlLabels: String (JSON array of associated ML Kit labels)
       - val confidenceThreshold: Float (default 0.9 per user decision)
       - val createdAt: Long

    2. **FolderEntity** - Represents discovered folders in Pictures/:
       - @PrimaryKey val uri: String (Document URI)
       - val name: String (folder name)
       - val displayName: String (user-friendly name)
       - val photoCount: Int
       - val isActive: Boolean (included in organization)
       - val learnedLabels: String (JSON of aggregated labels from 50 sample photos)
       - val learningStatus: String (PENDING/IN_PROGRESS/COMPLETED)
       - val createdAt: Long

    3. **PhotoMetadataEntity** - Tracks processed photos:
       - @PrimaryKey val uri: String (source photo URI)
       - val folderUri: String (which folder it belongs to)
       - val fileName: String
       - val fileSize: Long
       - val processedAt: Long?
       - val detectedLabels: String (JSON array of ML labels with confidence)
       - val status: String (PENDING/PROCESSED/FAILED/SKIPPED_LOW_CONFIDENCE)
       - val targetCategoryId: String?

    4. **FileOperationEntity** - Transaction log for crash recovery:
       - @PrimaryKey val id: String (UUID)
       - val sourceUri: String
       - val destUri: String
       - val operationType: OperationType enum (COPY, VERIFY, DELETE)
       - val status: OperationStatus enum (PENDING, COPYING, VERIFYING, DELETING, COMPLETED, FAILED)
       - val createdAt: Long
       - val completedAt: Long?
       - val retryCount: Int
       - val errorMessage: String?

    Create OperationType and OperationStatus enums in a separate file.
    Use kotlinx.serialization for JSON fields (not org.json).
  </action>
  <verify>
    Run: `./gradlew kaptDebugKotlin` - should compile without errors
    Check: Build output shows Room schema exported to app/schemas/
  </verify>
  <done>
    All four entity classes compile, Room processor generates schema, enums defined with proper converters
  </done>

<task type="auto">
  <name>Task 2: Create DAOs and Database class</name>
  <files>
    app/src/main/java/com/example/photoorganizer/data/local/database/dao/CategoryDao.kt

  <action>
    Create DAOs with Flow-based reactive queries and the Database class:

    1. **CategoryDao**:
       - @Query("SELECT * FROM categories") fun getAll(): Flow<List<CategoryEntity>>
       - @Query("SELECT * FROM categories WHERE id = :id") suspend fun getById(id: String): CategoryEntity?
       - @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun insert(category: CategoryEntity)
       - @Delete suspend fun delete(category: CategoryEntity)
       - @Query("SELECT COUNT(*) FROM categories") suspend fun getCount(): Int

    2. **FolderDao**:
       - @Query("SELECT * FROM folders") fun getAll(): Flow<List<FolderEntity>>
       - @Query("SELECT * FROM folders WHERE learningStatus = :status") fun getByLearningStatus(status: String): Flow<List<FolderEntity>>
       - @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun insertAll(folders: List<FolderEntity>)
       - @Update suspend fun update(folder: FolderEntity)
       - @Query("DELETE FROM folders") suspend fun deleteAll()
       - @Transaction suspend fun syncFolders(folders: List<FolderEntity>) { deleteAll(); insertAll(folders) }

    3. **PhotoMetadataDao**:
       - @Query("SELECT * FROM photo_metadata") fun getAll(): Flow<List<PhotoMetadataEntity>>
       - @Query("SELECT * FROM photo_metadata WHERE status = :status") fun getByStatus(status: String): Flow<List<PhotoMetadataEntity>>
       - @Query("SELECT * FROM photo_metadata WHERE uri = :uri") suspend fun getByUri(uri: String): PhotoMetadataEntity?
       - @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun insert(photo: PhotoMetadataEntity)
       - @Update suspend fun update(photo: PhotoMetadataEntity)
       - @Query("SELECT COUNT(*) FROM photo_metadata WHERE uri = :uri") suspend fun exists(uri: String): Int

    4. **FileOperationDao**:
       - @Query("SELECT * FROM file_operations WHERE status IN ('PENDING', 'COPYING', 'VERIFYING', 'DELETING')") fun getPending(): Flow<List<FileOperationEntity>>
       - @Query("SELECT * FROM file_operations WHERE status IN ('PENDING', 'COPYING', 'VERIFYING', 'DELETING')") suspend fun getPendingSync(): List<FileOperationEntity>
       - @Insert suspend fun insert(operation: FileOperationEntity)
       - @Update suspend fun update(operation: FileOperationEntity)
       - @Query("DELETE FROM file_operations WHERE createdAt < :cutoff AND status = 'COMPLETED'") suspend fun cleanupCompleted(cutoff: Long)

    5. **Converters** for enum and JSON:
       - OperationType converter (String ↔ OperationType)
       - OperationStatus converter (String ↔ OperationStatus)
       - Instant converter (Long ↔ java.time.Instant)

    6. **AppDatabase** abstract class:
       - @Database(entities = [CategoryEntity::class, FolderEntity::class, PhotoMetadataEntity::class, FileOperationEntity::class], version = 1)
       - abstract fun categoryDao(): CategoryDao
       - abstract fun folderDao(): FolderDao
       - abstract fun photoMetadataDao(): PhotoMetadataDao
       - abstract fun fileOperationDao(): FileOperationDao
       - Include type converters in @TypeConverters annotation
  </action>

    Check: Room compiler generates AppDatabase_Impl.java without errors
  </verify>

    Create Proto DataStore for user preferences per Now in Android pattern:

    1. **Create proto file** (app/src/main/proto/user_preferences.proto):
    ```protobuf
    syntax = "proto3";
    option java_package = "com.example.photoorganizer.data.local.datastore";
    option java_multiple_files = true;

    message UserPreferences {
      float confidence_threshold = 1;  // Default 0.9 (High)
      bool onboarding_completed = 2;
      repeated string folder_uris = 3;
      int32 learning_sample_size = 4;  // Default 50
      bool first_launch = 5;           // Default true
    }
    ```

    2. **Create UserPreferencesSerializer**:
    - Implement Serializer<UserPreferences>
    - defaultValue returns UserPreferences.getDefaultInstance()
    - Handle InvalidProtocolBufferException with CorruptionException

    3. **Create UserPreferencesRepository**:
    - Inject DataStore<UserPreferences>
    - Expose userData: Flow<UserData> mapping proto to domain model
    - Methods: setConfidenceThreshold(), setOnboardingCompleted(), addFolderUri(), removeFolderUri(), setFirstLaunchComplete()
    - Create UserData domain data class

    4. **Create DataStoreModule** (Hilt module):
    - @Provides @Singleton fun provideDataStore with DataStoreFactory.create()
    - Use produceFile = { context.dataStoreFile("user_prefs.pb") }
