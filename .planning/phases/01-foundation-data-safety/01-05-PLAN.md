  <files>
    app/src/main/java/com/example/photoorganizer/data/repository/LearningRepository.kt

  <action>
    Create repository for managing learning process:

    1. **LearningRepository class**:
       - @Inject constructor with folderDao, mlLabelingDataSource, safDataSource
       - suspend fun startLearning(folderUri: String): Result<LearningResult>
       - suspend fun getPendingFolders(): List<FolderEntity> - folders with PENDING status
       - suspend fun updateFolderLearningStatus(uri: String, status: LearningStatus)
       - suspend fun saveLearningResult(result: LearningResult)

    2. **Learning algorithm** (per user decision - combined approach):
       - Sample 50 photos from folder (random selection, not sequential)
       - For each photo: analyze with ML Kit, collect labels
       - Aggregate labels across all samples:
         - Count frequency of each label
         - Calculate average confidence per label
       - Boost folder name match (if folder name contains label text, boost confidence)
       - Store top 10 labels in learnedLabels field

    3. **Sampling logic**:
       - Get all photo URIs from folder via SafDataSource
       - If < 50 photos, use all available
       - Shuffle and take first 50
       - Track progress: "Analyzing photo X of 50"

    4. **Label aggregation**:
       - Map<labelText, Pair<count, sumConfidence>>
       - After all samples: calculate avg confidence for each label
       - Sort by count, then by confidence
       - Store top 10 as JSON in folder entity
  </action>

    Check: Repository compiles with all dependencies
  </verify>

  <action>
    Create WorkManager worker for background learning:

    1. **FolderLearningWorker class**:
       - Extend CoroutineWorker(context, params)
       - @HiltWorker with @AssistedInject constructor
       - Inject LearningRepository, FolderRepository
       - override suspend fun doWork(): Result

    2. **Worker implementation**:
       ```
       - Get pending folders from repository
       - For each pending folder:
         - Update status to IN_PROGRESS
         - Call learningRepository.startLearning(folderUri)
         - On success: update status to COMPLETED
         - On failure: retry if retryCount < 3
       - Return Result.success() or Result.retry()
       ```

    3. **Progress tracking**:
       - Use setProgressAsync() to report progress
       - Data with keys: "folderName", "current", "total"
       - UI can observe progress via WorkManager

    4. **Worker constraints** (per research - battery aware):
       - setRequiresBatteryNotLow(true)
       - setRequiresStorageNotLow(true)
       - setBackoffCriteria(EXPONENTIAL, 10s, 5min)

    5. **WorkerModule** (Hilt):
       - @Module @InstallIn(SingletonComponent::class)
       - abstract fun bindWorkerFactory
       - Configure WorkerManager with HiltWorkerFactory

    6. **Scheduling**:
       - Schedule from onboarding completion
       - OneTimeWorkRequest for immediate learning
       - Unique work name to prevent duplicates
  </action>
